# Product Vision

## POTOKI Is

- calm,
- focused,
- local,
- context-oriented,
- technical,
- quiet,
- premium,
- minimal,
- privacy-first.

## POTOKI Is Not

- a task manager,
- a Notion clone,
- a Jira clone,
- enterprise SaaS,
- a productivity dashboard,
- gamified productivity software,
- a time tracker,
- a team management system.

## Target User

Technical user with many interrupted parallel contexts:

- refactors,
- architecture reviews,
- research threads,
- testing work,
- documentation work,
- operational notes.

Often works in restricted corporate/banking environments where cloud tools may be unavailable or not allowed.

## Desired Experience

The user should feel:

- "I can return to this thought."
- "My context is still here."
- "The app is quiet enough for deep work."
- "My files are under my control."

The user should not feel:

- managed,
- scored,
- watched,
- nudged into productivity rituals.

## Inspirations

- Obsidian
- Linear
- Raycast
- Arc
- local-first apps
- desktop-native UX

## Product Direction

- Continuity over productivity.
- Memory over task management.
- Local files over cloud systems.
- Reading-first context over form-heavy editing.
- Subtle interaction over loud UI.
