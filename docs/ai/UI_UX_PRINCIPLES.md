# UI/UX Principles

## Visual Language

- Dark premium UI.
- Graphite/navy layered surfaces.
- Muted turquoise/cyan accent.
- Typography-first layout.
- Low visual noise.
- Soft surface depth.

## Layout

- Three-panel workspace shell.
- Collapsible left sidebar for focus.
- Center content is primary.
- Right context rail is ambient support.
- Use whitespace and rhythm over heavy boxes.
- Avoid dashboard widget feeling.

## Color

- Cyan is an accent, not decoration.
- Prefer tinted surfaces over glow.
- Avoid neon, cyberpunk, RGB, gaming effects.
- Use contrast carefully; low-noise does not mean unreadable.

## Motion

- Subtle, restrained, almost invisible.
- Use opacity, transform, soft easing.
- No playful bounce.
- No flashy effects.
- Flow indicator should feel ambient, not like a custom scrollbar gimmick.

## Typography

- Reading comfort matters.
- Body text should not be too muted.
- Metadata should be quiet.
- Headings should anchor sections without shouting.

## Interaction

- Prefer lightweight inline editing.
- Destructive actions require confirmation.
- Notes/files open in modal overlays, not inline timeline takeover.
- Toolbar controls should be compact, consistent, keyboard accessible.

## Avoid

- Enterprise admin panels.
- KPI cards.
- Heavy borders.
- Giant CTA buttons.
- Glassmorphism overload.
- Strong glow.
- Too many badges.
- Dense form-like right panels.
- Activity tracking dashboards.

## Accessibility Expectations

- Buttons need `aria-label` when icon-only.
- Focus visible must exist.
- Keyboard workflows should remain intact.
- Do not hide functionality behind hover only if it becomes inaccessible.
